프로젝트 준비 순서
1) 요구사항 분석
2) 리액트 앱 만들기
3) 폰트 설정하기
4) 이미지 준비하기

1. 요구사항 분석
[감정 일기장]은 일기를 작성하면서 그날의 자기 감정을 표현하는 서비스
감정일기장은 모두 4페이지로 구성되어 있음

Home 페이지)
- 사용자가 앱에 접속하면 처음으로 만나는 페이지
- 인덱스 페이지라고도 부르며 웹서비스 대문 역할
- 경로: '/'
- 상단 헤더 섹션과 하단 일기 리스트로 구성
- 헤더 섹션에는 월 단위로 일기를 조회하는 기능이 있는데, 좌우 버튼을 클릭하면 월 단위로 날짜를 이동
- 일기 리스트 섹션에는 '최신순' '오래된 순'으로 일기 리스트를 정렬하는 기능과 새로운 일기를 추가하는 기능이 있음
- <새 일기 쓰기> 버튼을 클릭하면 새로운 일기를 작성하는 New 페이지로 이동

New 페이지)
- 새로운 일기를 추가하는 페이지
- 경로: '/new'
- <뒤로 가기> 버튼이 있는 헤더와 날짜 입력, 감정 이미지 선택, 일기 작성 폼으로 구성
- 페이지 하단에는 <취소하기>, <작성 완료> 버튼
- <작성완료> 버튼을 클릭하면 Home으로 되돌아감

Diary 페이지)
- 특정한 일기를 상세히 조회하는 페이지
- Home에서 일기 리스트를 조회한 다음, 특정 일기를 클릭하면 Diart 페이지로 이동
- 경로: '/diary/(일기)id'
- Diary 페이지 헤더에는 <뒤로 가기>와 <수정하기> 버튼이 있음
- <수정하기> 버튼을 클릭하면 Edit 페이지로 이동

Edit 페이지)
- 작성한 일기를 수정하는 페이지.
- Home 또는 Diary 페이지에서 <수정하기> 버튼을 클릭하면 Edit 페이지로 이동
- 경로: '/edit/(일기)id'
- New 페이지와 모습이 유사하지만, 헤더의 제목이 다르고 <삭제하기> 버튼이 추가되었다.
